1 Overview

Security Hardening is a collection of skills for Claude focused on security code review, hardening patterns, vulnerability detection, and secure coding best practices. It brings structured security analysis directly into the development workflow.

The skills cover offensive and defensive perspectives - identifying vulnerabilities, applying fixes, and auditing dependencies for known issues.

2 Features

  • OWASP Top 10 checks - systematic review against the most critical web application risks
  • Input validation - detect missing or weak validation that opens injection paths
  • SQL injection detection - identify unsafe query construction and parameterisation gaps
  • XSS prevention - find unescaped output, unsafe innerHTML usage, and CSP gaps
  • Secure dependency audits - flag outdated packages with known CVEs
  • Hardening patterns - recommend security headers, least-privilege configs, and safe defaults

3 Coverage

Category Checks included
Injection SQL, NoSQL, command, LDAP injection patterns
XSS Reflected, stored, DOM-based - output encoding gaps
Authentication Session management, token storage, brute force exposure
Authorization IDOR, privilege escalation, missing access control checks
Dependencies CVE matching, outdated packages, supply chain risks
Configuration Security headers, CORS policy, TLS settings, secrets exposure

4 Installation

  1. Open Claude Code
  2. Run /skills install security-hardening
  3. The skills activate in the current session
  4. Use /security-hardening to trigger a review on the current file or selection
Note: Skills work on any language. Point them at a file, a function, or a full codebase path.

5 Requirements

  • Claude Code with skill support
  • No external tooling required - analysis runs through Claude
  • For dependency audits: a package.json, requirements.txt, or equivalent manifest
