1 Overview

gemini-mcp-connect is a Python MCP server that bridges Claude Code with Google Gemini, acting as a second-opinion agent directly inside your AI CLI. It lets you validate plans, review code, and cross-check reasoning without leaving your terminal.

The server exposes four tools accessible from Claude Code: free-form queries, code review, plan validation, and quota status. Three reasoning tiers control the trade-off between speed and depth.

2 Features

  • Second-opinion agent - ask Gemini from inside Claude Code
  • Three reasoning tiers - lite, flash, pro for different use cases
  • Code review tool - language-aware critique via review_code
  • Plan validation - validate architecture or approach before execution
  • Quota tracking - check remaining daily requests with gemini_status
  • Zero-friction install - run with uvx, no clone required

3 Tools

The server exposes four MCP tools:

Tool Parameters Description
query_gemini prompt, tier Free-form question to Gemini
review_code code, language, tier Language-aware code review
validate_plan plan, tier Validate a plan before execution
gemini_status - Check remaining daily quota

Reasoning tiers

Tier Speed Use case
lite Fast Quick lookups, low-stakes queries
flash Balanced General code review and validation
pro Max reasoning Complex architecture, critical decisions (100 req/day)

4 Installation

Option 1 - uvx (recommended)

No clone needed. Run directly with uv:

uvx gemini-mcp-connect

Option 2 - From source

Requires Python 3.10+.

git clone https://github.com/StealthyLabsHQ/gemini-mcp-connect.git
cd gemini-mcp-connect
pip install -e .

5 Configuration

Set your Google AI Studio API key as an environment variable: 

export GEMINI_API_KEY=your_api_key_here

Add the server to your Claude Code MCP configuration file (~/.claude/settings.json or equivalent): 

{
  "mcpServers": {
    "gemini-mcp-connect": {
      "command": "uvx",
      "args": ["gemini-mcp-connect"],
      "env": {
        "GEMINI_API_KEY": "your_api_key_here"
      }
    }
  }
}
Note: If installing from source, replace "uvx" with "python" and adjust args to point to the module entry point.
