1 Presentation

Security Hardening est une collection de skills pour Claude axes sur la revue de securite du code, les patterns de durcissement, la detection de vulnerabilites et les bonnes pratiques de code securise. Il integre une analyse de securite structuree directement dans le flux de developpement.

Les skills couvrent les perspectives offensives et defensives : identification des vulnerabilites, application des correctifs et audit des dependances pour les problemes connus.

2 Fonctionnalites

  • Verifications OWASP Top 10 - revue systematique contre les risques les plus critiques des applications web
  • Validation des entrees - detection des validations absentes ou insuffisantes qui ouvrent des chemins d'injection
  • Detection d'injection SQL - identification des constructions de requetes non securisees et des lacunes de parametrisation
  • Prevention XSS - identification des sorties non echappees, utilisation non securisee d'innerHTML et lacunes CSP
  • Audit des dependances - signalement des paquets obsoletes avec des CVE connus
  • Patterns de durcissement - recommandations d'en-tetes de securite, configurations avec moindre privilege et valeurs par defaut securisees

3 Perimetre de couverture

Categorie Verifications incluses
Injection SQL, NoSQL, commande, patterns d'injection LDAP
XSS Reflechi, stocke, base DOM - lacunes d'encodage des sorties
Authentification Gestion de session, stockage des tokens, exposition aux attaques par force brute
Autorisation IDOR, elevation de privileges, controles d'acces manquants
Dependances Correspondance CVE, paquets obsoletes, risques de chaine d'approvisionnement
Configuration En-tetes de securite, politique CORS, parametres TLS, exposition de secrets

4 Installation

  1. Ouvrir Claude Code
  2. Executer /skills install security-hardening
  3. Les skills s'activent dans la session en cours
  4. Utiliser /security-hardening pour declencher une revue sur le fichier ou la selection en cours
Note : Les skills fonctionnent avec tous les langages. Pointez-les vers un fichier, une fonction ou le chemin complet d'une base de code.

5 Prerequis

  • Claude Code avec support des skills
  • Aucun outil externe requis - l'analyse s'effectue via Claude
  • Pour les audits de dependances : un manifeste package.json, requirements.txt ou equivalent
