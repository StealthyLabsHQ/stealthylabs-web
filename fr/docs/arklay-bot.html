Arklay
Discord Bot

Retour aux docs

1 Apercu

Arklay est un bot Discord open-source modulaire construit avec TypeScript et discord.js v14. Il propose 99 commandes slash reparties en 6 modules independants : lecteur musical, assistant IA, moderation, utilitaires, fun et configuration serveur.

La musique est propulsee par Lavalink + Shoukaku avec SoundCloud comme source principale. L'IA supporte 4 fournisseurs : Claude (Anthropic), Gemini (Google), ChatGPT (OpenAI) et des modeles locaux via Ollama. Toutes les donnees sont persistees via SQLite. Supporte les commandes slash et le prefixe texte (.commande ou arklay commande). Repondre a un message IA continue la conversation.

Disponible sur github.com/StealthyLabsHQ/arklay-bot.

2 Modules

2.1 Musique (27 commandes)

Lecteur musical Lavalink + Shoukaku supportant YouTube, Spotify et SoundCloud. Embed "En cours de lecture" persistant avec boutons interactifs et menu de filtres. La file reprend automatiquement apres redemarrage. Deconnexion automatique apres 5 minutes d'inactivite. SoundCloud est la source principale, YouTube en secours. Si Lavalink n'est pas lance, les commandes musicales sont desactivees avec un message clair.

3 Fonctionnalites automatiques

Comportements evenementiels ne necessitant aucune commande :

  • Auto-role - attribue un role a chaque nouveau membre si configure via /config autorole
  • Messages de bienvenue - accueille les nouveaux membres avec un texte personnalisable (variables {user} et {server})
  • Salons vocaux temporaires - rejoindre un salon hub cree un salon personnel, supprime automatiquement quand vide
  • Journalisation des messages - messages supprimes et modifies enregistres dans le salon de logs
  • Auto-moderation IA - analyse de contenu Gemini optionnelle (anti-rebond), enregistre le contenu signale
  • Reprise musicale automatique - la file est automatiquement restauree apres un redemarrage du bot
  • Deconnexion automatique - le bot quitte le vocal apres 5 minutes d'inactivite
  • Suivi des sniped messages - dernier message supprime et modifie suivi par salon

4 Commandes par prefixe texte

Toutes les commandes fonctionnent avec un prefixe texte en plus des commandes slash :

  • Format : .commande [args] ou arklay commande [args]
  • Prefixe par defaut : . (configurable via BOT_PREFIX dans .env)
  • Nom du bot par defaut : arklay (configurable via BOT_NAME dans .env)
  • Exemple : .play never gonna give you up ou arklay ask what is rust
  • Repondre a un message IA continue la conversation
  • Note : les groupes de sous-commandes complexes (/config, /warn, /botrole, /nanobanana) ne sont pas disponibles en prefixe texte

5 Installation

Demarrage rapide

git clone https://github.com/StealthyLabsHQ/arklay-bot.git
cd arklay-bot
npm install
cp .env.example .env
# Editer .env - ajouter au minimum DISCORD_TOKEN et CLIENT_ID
npm run deploy
npm run dev

Le bot fonctionne sans Lavalink - les commandes musicales affichent "Lavalink server is not running". Toutes les autres fonctionnalites sont independantes.

Lavalink (requis pour la musique)

La musique est propulsee par Lavalink, un serveur audio independant. Java 17+ est requis.

Installer Java :

  • Ubuntu/Debian : sudo apt install openjdk-17-jre-headless -y
  • Windows : telecharger depuis adoptium.net (Temurin 21 LTS), activer "Add to PATH" lors de l'installation
  • macOS : brew install openjdk@17

Installer et lancer Lavalink :

mkdir lavalink && cd lavalink
wget https://github.com/lavalink-devs/Lavalink/releases/latest/download/Lavalink.jar
cp ../lavalink/application.yml .
java -jar Lavalink.jar

En production avec PM2 :

pm2 start "java -jar Lavalink.jar" --name lavalink --cwd /path/to/lavalink
pm2 save

Installation complete

  1. Cloner le depot : 
    git clone https://github.com/StealthyLabsHQ/arklay-bot.git
cd arklay-bot
  2. Installer les dependances : 
    npm install
  3. Copier le fichier d'environnement : 
    cp .env.example .env
  4. Editer .env avec vos tokens et cles API (voir la section Configuration ci-dessous).
  5. Deployer les commandes slash sur Discord : 
    npm run deploy
  6. Demarrer le bot :

    Mode developpement :

    npm run dev

    Ou compiler et lancer en production :

    npm run build
npm run start

Production avec PM2

npm run build
npm run deploy
pm2 start dist/index.js --name arklay-bot
pm2 save

Configuration du portail Discord Developer

  • Aller sur discord.com/developers/applications et creer une application
  • Dans l'onglet Bot, activer : Server Members Intent, Message Content Intent, Presence Intent
  • Dans OAuth2, generer une URL d'invitation avec les scopes : bot, applications.commands
  • Permissions recommandees : Administrator (ou specifiques : Manage Roles, Manage Channels, Kick/Ban Members, Moderate Members, Manage Messages, Connect, Speak)

Optionnel : placer un fichier cookies.txt (format Netscape) a la racine du projet pour contourner la detection de bots YouTube lors de la lecture musicale.

6 Configuration

Toute la configuration se fait via les variables d'environnement dans le fichier .env.

Requises

Variable Description
DISCORD_TOKENToken du bot depuis le portail Discord Developer
CLIENT_IDID de l'application depuis le portail Discord Developer

Optionnelles

Variable Description
GUILD_IDID du serveur de dev pour deploiement instantane (sans GUILD_ID = global, jusqu'a 1 heure)
LAVALINK_HOSTAdresse du serveur Lavalink (defaut : localhost:2333)
LAVALINK_PASSWORDMot de passe Lavalink (defaut : youshallnotpass)
LAVALINK_SECUREUtiliser WSS au lieu de WS (defaut : false)
BOT_PREFIXPrefixe des commandes texte (defaut : ".")
BOT_NAMEPrefixe nom du bot pour les commandes texte (defaut : "arklay")
BOT_OWNER_IDID Discord du proprietaire (obtient des limites IA elevees)
BOT_OWNER_MULTIPLIERMultiplicateur de limite proprietaire 0-20 (defaut : 5)
ANTHROPIC_API_KEYActive les commandes IA Claude
GOOGLE_AI_API_KEYActive l'IA Gemini + generation d'images /nanobanana
OPENAI_API_KEYActive les modeles ChatGPT (GPT-5.4 Nano/Mini, Codex, o4 Mini)
GIPHY_API_KEYActive la commande /gif (gratuit sur developers.giphy.com)
OLLAMA_ENABLEDActiver/desactiver l'IA locale (true/false, defaut : false)
OLLAMA_HOSTAdresse du serveur Ollama (defaut : http://localhost:11434)
OLLAMA_MODELModele IA local (defaut : gemma4:26b)
OLLAMA_KEEP_ALIVEDuree de maintien du modele en RAM apres la derniere requete (defaut : 5m)
SPOTIFY_CLIENT_IDResolution des liens Spotify
SPOTIFY_CLIENT_SECRETResolution des liens Spotify

Au moins un fournisseur IA (Anthropic, Google, OpenAI ou Ollama) est necessaire pour le module IA. Si aucun n'est configure, le module IA se desactive automatiquement.

7 Prerequis

  • Node.js 20+ (teste sur Node 24)
  • Java 17+ et Lavalink (pour la lecture musicale - optionnel, le bot fonctionne sans)
  • Token de bot Discord (obligatoire)
  • Au moins une cle API IA pour utiliser les commandes IA (Anthropic, Google, OpenAI ou Ollama)
